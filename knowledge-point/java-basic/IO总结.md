关于文件操作的总结：
1.File:只能操作文件/目录的相关属性（周边信息），操作不了文件的内容

2.RandomAccessFile：基于指针的操作，能够随机（对指针的操作）访问文件
    的内容；
	读和写都是：低八位，读的是当前指针指向的字节位置，读完后，指针自动后移
	read():读一个字节
	write():写一个字节
	为了提高读写效率，创建一个字节数组
		byte[] buf=new byte[字节长度]；
			read(buf);
			write(buf):不常用
			要读的字节数组，从字节数组的哪个位置开始，读多少字节数
			write(buf,0,length):常用
	关于RandomAccessFile类一些指针操作的方法：
		getPointFile():获取指针的当前位置
		seek(long l):任意跳到对应的字节位置
		skipBytes(int i):向后跳过多少字节
-------------------------------------------------
3.IO流
	低级流（节点流）：有固定来源或者去向的流
	高级流（处理流）：建立在低级流的基础之上，对流的操作进行封装，来处理低级流
		 高级流可以叠加使用
	
	字节流
	对象流
	字符流
		 
------------------------字节流--------------------
字节流的抽象类：
	InputStream:字节输入流的父类
	OutputStream:字节输出流的父类
	第一组实现流：低级流
	FileInputStream：文件字节输入流
	FileOutputStream：文件字节输出流
	--内容的追加
	read():读一个字节
	write():写一个字节
	为了提高读写效率，创建一个字节数组
		byte[] buf=new byte[字节长度]；
			read(buf);
			write(buf):不常用
			要读的字节数组，从字节数组的哪个位置开始，读多少字节数
			write(buf,0,length):常用
	这两个流和RandomAccessFile区别
	1.方法一致
	2.用法一致
	3.在进行写的过程中FileOutputStream可以使用没有使用追加的功能，
		会将原文件中的所有内容清除，再重新写入
		RandomAccessFile如果写的字节数大于原文件，则全部覆盖
		如果写入的字节数小于原文件的字节数，则只覆盖原文件中与写入字
		节数等量的内容
	第二组实现流：高级流
	BufferedInputStream：缓冲字节输入流
	BufferedOutputStream：缓冲字节输出流
	这两个类各自内部维护这一个缓冲区，在数据写出时，如果缓冲区未满，则
	数据不会主动写出，若需要将未满的缓冲区数据强制写出，则需要调用
	flush方法。
	使用这两个类的对象时，需要先创建一个低级流对象，供其操作
	read():
	write():
	flush():强制将未满的缓冲区数据写出，会增加读写次数，降低读写效率
	第三组实现流：高级流（对象流）
	ObjectInputStream：对象输入流
	ObjectOutputStream：对象输出流
	对象--->字节：序列化
	字节--->对象：反序列化
	serializable接口：如果想将某个对象实现序列化，那么该对象的类
	必须实现该接口，则才有资格做可序列化操作
	序列化版本号：为了解决后期的变更需求，导致类的属性/方法修改，在
	反序列化的时候出现异常，引入版本号；
	transient:它修饰的成员变量的值，在序列化时不会被保存。
	readObject():一次读取一个对象的字节，返回值为Object类型，需要
				进行强制类型转换，才能调用对应类型的方法和属性
	writeObject(Object obj):一次写一个对象的字节，无返回值
-------------------字符流-----------------------------------
字符流的两个抽象的父类:（转换流       字节流--->字符流(将字节转换成字符处理)）
	字符流可以设置字符的编码格式
	Reader：	字符输入流（父类）
	Writer:	字符输出流（父类）
	注意判断字节流和字符流：以Reader/Writer结尾都是字符流
	字符流只能操作文本文件，不能操作图片，视频，音频......
	(字节流可以操作任何形式的数据)
	实现类：
	InputStreamReader：字符输入流
	OutputStreamWriter：字符输出流
	read():每次读一个字符（根据对应的字符集编码读取多少个字节）
	write():每次写一个字符（根据对应的字符集编码写入多少个字节）
	缓冲字符流：
	BufferedReader:(BufferedWriter不好用):缓冲字符输入流
		readLine():每次读取一整行数据，并返回，若没有数据可读
		则返回值null
	PrintWriter:缓冲字符输出流
		print():可以写任何类型的数据
		println():写出数据后自动换行，在使用new PrintWriter(pw,true)
		创建对象使用自动行刷新时，会自动将缓存区未满的数据写出
		
	所有涉及到IO的操作，都必须调用close（）方法
