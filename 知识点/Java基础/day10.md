接口：
面向对象的编程<面向接口编程
--以面向对象为核心的面向接口的编程
实例：USB接口

	电脑厂商做USB就相当于定义了一个规范，标准，只要遵循这个标准才能将设备插到电脑上使用

	1.接口就是一个标准，规范（目的是制定你要遵循的规范）--关键interface表示是一个接口
	2.接口中只能有常量和抽象方法
	3.接口不能创建对象（实例化），实现接口的类，叫做实现类，使用关键字implement实现接口
	4.接口支持多实现，，如果一个实现类有继承，且有实现的话，需要先继再实现
	5.实现类实现接口，需要将接口中所有的抽象方法实现
	6.接口支持继承，且支持多继承，多个接口之间用逗号隔开
	7.实现类中的权限修饰符要等于接口中的权限修饰符
接口和抽象类区别：
1.成员变量的区别:
	抽象类：
		1.成员变量可以是常量也可以是变量
		2.有构造方法
		3.成员方法可以是抽象也可以是非抽象
	接口：
		1.成员变量只能是常量
		2.没有构造方法
		3.成员方法只能是抽象方法

2.相同点：都不能被实例化，需要用多态来实现实例化


类与类：继承属性为主
抽象类和类：一般是继承部分方法和属性
类与接口：完全的功能实现

多态的意义：
1.一个类型的引用在指向不同的对象时会有不同的实现
   cut（）;---方法的多态
	1.理发师--cut() 剪发
	2.厨师--cut()  切菜
	3.医生---cut()  做手术
2.同一个对象造型成不同类型时有不同的功能
	我
	--老师   讲课
	--子女   孝敬 
	--孩子他爸   教育孩子
细节：
编辑器认为：父类大，子类小；接口大，实现类小
在引用类型进行强制类型转换时只看对象（对象是什么类型的就可以转换成什么类型）
转换的前提：一定存在继承或者实现的关系才能进行强制类型转换（前提-向上造型的写法）
instanceof：判断某个引用指向的兑现是否为指定类型（防止类型转换异常）--boolean类型	



内部类：
一个类可以定义在另外一个类里面，外部的类叫做外部类，里面的类叫做内部类
内部类定义在外部类里面，通常只服务于外部类
内部类可以使用外部类的所有成员变量和方法（包括私有的）



匿名内部类（重点）写法：
如果在一段程序中需要创建一类对象（这个类一般需要实现某个接口或者继承某个类 ），这个类可以不必命名，称之为匿名内部类
何时用：只需要一个对象，对象创建完毕后，这个类没有意义了


面向对象:

面向对象的三大特性:
1.封装：保证数据的安全   
类-封装数据，行为，作为一个整体
方法-封装功能的实现，隐藏实现的细节


2.继承：实现代码的重用
关键字：extends

3.多态：多种形态，在继承的基础之上
	提高代码的可维护性，可拓展性
1.一个类型引用指向不同对象时，有不同的实现
2.同一个对象造型成不同类型时，有不同的功能
